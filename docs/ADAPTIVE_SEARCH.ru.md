# Адаптивный поиск (opt-in, Pass Adaptive.1)

Адаптивный поиск выполняет **детерминированное расписание эскалации** из обычных
фиксированных проходов редукции и останавливается на первом *сертифицированном* `Success`.
Это **контролируемое расширение расписания, а не универсальное доказательство редуцируемости**:
исчерпание расписания означает лишь «ни одна из этих конкретных конфигураций не
сертифицировала редукцию» — и никогда не доказывает, что редукции не существует.

## Контракт честности

- Каждый уровень — это один обычный проход `reduce_family_once`. Адаптивный режим не
  добавляет **ни новой математики, ни нового пути к `Success`**: верификация реконструкции,
  сертификатный gate по строкам и проверка `AllLocallyFinite` применяются к каждому уровню
  ровно так же, как в одиночном фиксированном проходе.
- Пройденный сертификат при не локально-конечной или `Unknown` нормальной форме — по-прежнему
  типизированный отказ; адаптивный режим никогда не «повышает» его до успеха.
- Адаптивный режим **строго opt-in**. Без `--adaptive` (CLI) или адаптивного API поведение —
  неизменный одиночный фиксированный проход.
- RREF-бэкенд по умолчанию остаётся `dict`; уровень расписания может явно выбрать другой
  бэкенд (например `auto` — рекомендуется при установленной extra `speed`).

## Python API

```python
from parametric_ibp_lf_reducer import (
    AdaptiveSearchConfig, SearchLevel,
    reduce_family_adaptive, reduce_wolfram_style_input_adaptive,
    default_search_levels,
)

result = reduce_wolfram_style_input_adaptive(input_text)  # дефолтное 3-уровневое расписание
result = reduce_wolfram_style_input_adaptive(
    input_text,
    search=AdaptiveSearchConfig(
        levels=(
            SearchLevel(name="small", label_box=((0, 0), (-1, 0)), max_ibp_degree=1,
                        tangent_degree_blocks=()),
            SearchLevel(name="wide", label_box=((0, 0), (-2, 0)), max_ibp_degree=2,
                        tangent_degree_blocks=((1, 1),)),
        ),
        max_labels=20_000, max_rows=200_000, timeout_sec=600.0,
    ),
)
```

`reduce_family_adaptive(family, target_label, config, search=None)` — базовая точка входа для
уже распарсенных семейств. Поля `SearchLevel`, равные `None`, наследуются из базового
конфига; `tangent_degree_blocks=()` означает «на этом уровне явно без tangent-строк»;
`extra_samples` / `extra_primes` детерминированно **расширяют** (никогда не заменяют) базовые
scattered-точки и список простых.

## Дефолтное расписание (`default_search_levels`)

Строится только из базового конфига (базовый бокс = `LabelBox` из документа/override либо
пакетный дефолт `((0, 0), (-1, 0))`):

| уровень | имя | бокс меток | `max_ibp_degree` | tangent-блоки | дополнительно |
|---|---|---|---|---|---|
| 0 | `base` | базовый бокс | 1 | нет | — |
| 1 | `expand-1` | каждый m-диапазон углублён на 1 | 2 | `((1, 1),)` | — |
| 2 | `deep` | каждый m-диапазон углублён на 2 | 2 | `((1, 1), (2, 2))` | +4 точки, +2 простых; опционально `rref_backend` (рекомендуется `"auto"`) |

n-диапазоны никогда не меняются. Явный список `labels` детерминированно расширить нельзя —
задавайте явные `SearchLevel` (иначе `ValueError`).

## Результат и история

Возвращается обычный `ReductionResult`:

- первый сертифицированный `Success`, либо
- **лучший частичный отказ**, если ни один уровень не успешен; порядок детерминирован:
  Success → цель редуцирована → меньше не-LF термов → меньше `Unknown` термов → реконструкция
  верифицирована → качество сертификата (`Passed` > `Insufficient` > `NotRun` > `Failed`) →
  более ранний уровень (tie-break).

Полная история — в `result.diagnostics.extra["adaptive"]` (и в CLI-`--diagnostics-json` под
ключом `"adaptive"`): статус каждого уровня, причина отказа, статус сертификата, счётчики
строк/меток/записей, **рекомендация** по типу отказа (например `TargetNotReducible` →
расширить бокс/степени/tangent; `InterpolationFailed` → больше точек/простых;
`VerificationFailed` → добавить независимые точки и **никогда** не принимать текущие
коэффициенты), а также wall-clock тайминги (только для наблюдаемости).

## Лимиты ресурсов

| лимит | семантика |
|---|---|
| `max_levels` | усекает расписание (дефолт 3) |
| `max_labels` | **предпроверка**: слишком большой уровень *не запускается*; если не запустился ни один — типизированный отказ `ResourceLimitReached` |
| `max_rows` | проверяется **после** завершения уровня; останавливает дальнейшую эскалацию (завершённый уровень засчитывается) |
| `timeout_sec` | проверяется **только между** уровнями; запущенный уровень никогда не прерывается |

Каждое срабатывание лимита отражается точно (`kind`, `limit`, `observed`, `level`) в
`extra["adaptive"]["resource_limit"]`. `timeout_sec` — единственная wall-clock-ручка,
по умолчанию выключена и никогда не меняет математическое содержимое завершённых уровней.

## CLI

```
python -m parametric_ibp_lf_reducer reduce input.m --adaptive
python -m parametric_ibp_lf_reducer reduce input.m --adaptive --adaptive-max-levels 2 \
    --diagnostics-json diag.json
```

`--adaptive-max-levels` требует `--adaptive`; без `--adaptive` путь CLI байт-в-байт совпадает
с прежним одиночным проходом. Коды выхода не изменились: `0` только для сертифицированного
`Success` (какого-либо уровня), `1` — честный отказ, `2` — ошибка использования.
